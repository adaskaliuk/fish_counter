# Acceptance criteria / Критерії прийняття

- Result screen shows a readable timeline of the match.
- User can see what happened in each time window.
- User can see which button/event was recorded and when.
- User can enter/edit final weight, final count, or both.
- User can see final result values in results/history/export.
- User can see weather during the match, not only final/current weather.
- Temperature, pressure, wind, humidity are visible when available.
- Weather/activity relationship is summarized in plain language.
- Sessions without weather data still show timeline and totals.
- No crashes on old sessions or partial weather data.
